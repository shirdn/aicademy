# Force Plan Mode

**STOP. Do not write any code yet.**

Before implementing anything, you must create a plan and get approval.

## Planning Steps

1. **Understand the request**
   - What is being asked?
   - What are the acceptance criteria?
   - What's unclear? (ask the user)

2. **Read the codebase**
   - What exists already?
   - What patterns are in use?
   - What would be affected by this change?

3. **Design the approach**
   - What files need to change?
   - What new files are needed?
   - What's the simplest approach that works?

4. **Identify risks**
   - What could go wrong?
   - What are the dependencies?
   - Any irreversible changes?

5. **Present the plan**
   Format:
   ```
   ## Plan: [Feature Name]

   **Goal:** [What we're building]

   **Approach:**
   1. [Step 1]
   2. [Step 2]
   3. [Step 3]

   **Files to change:**
   - `path/to/file` — [what changes]

   **New files:**
   - `path/to/new/file` — [purpose]

   **Risks:**
   - [Risk 1]
   - [Risk 2]

   **Tests needed:**
   - [Test 1]
   - [Test 2]
   ```

6. **Wait for approval**
   - Do NOT proceed until the user says "go" or "approved"
   - If they have feedback, update the plan first

**Remember:** Planning is not wasted time. A 10-minute plan saves hours of rework.
